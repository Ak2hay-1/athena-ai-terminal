# Security Policy

## Supported Versions

| Version | Supported |
|----------|-----------|
| 1.x | ✅ |
| 0.x-dev | Development Only |

---

## Reporting Security Issues

Please do not create public GitHub issues for security vulnerabilities.

Instead, report vulnerabilities privately to the project maintainers.

Include:

- Description
- Steps to reproduce
- Impact assessment
- Suggested mitigation (optional)

---

## Security Practices

Athena follows these principles:

- No secrets committed to Git
- Environment-based configuration
- JWT authentication
- Principle of least privilege
- Input validation
- SQLAlchemy ORM
- HTTPS in production
- Dependency updates
- Security reviews before releases
